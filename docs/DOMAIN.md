# Vocare Courses — Domain

## The Mental Model

Vocare Courses is a learning platform where paying customers receive an access token to log in and take a phone sales training course. There are no usernames or passwords — a token IS the credential. Payment happens externally; the platform only manages access and learning.

---

## Core Models

### AdminUser (Devise)
Platform administrators who manage tokens, customers, and course content. Created via Rails console. Multiple admins supported.

### Token
The login credential. Format: `VOCARE-XXXX-XXXX`. Generated by admins, given to paying customers. A token can be unused, active (linked to a customer), expired, or revoked.

### Customer
A person taking the course. Created on first token use (enters name, optional email/phone/company). Called "Customer" not "Student" because these are paying professionals. The model is extensible for future profile data.

### Course
The top-level container. MVP has one course: "Vocare Salgskursus". Has a slug, description, and published flag.

### Section (Chapter)
A chapter within a course. Ordered by position. Contains lessons and one quiz. Sections are locked until the previous section's lessons are completed and quiz is passed. Section 1 is always unlocked.

### Lesson
A piece of content within a section. Three types:
- **video** — Vimeo embed URL
- **audio** — MP3 file (Active Storage)
- **text** — Rich text body

Ordered by position within section.

### Quiz
Attached to a Section (section quiz) or Course (final quiz) via polymorphic `quizzable`. Has a passing score (default 70%). Contains ordered questions.

### QuizQuestion
A free-text question with an expected answer (reference for AI comparison). No multiple choice — students write their answer in their own words.

### QuizAttempt
A student's attempt at a quiz. Contains answers, overall score, pass/fail. Unlimited retakes allowed. Evaluated asynchronously by OpenAI.

### QuizAnswer
One answer within an attempt. Contains the student's free-text response, AI evaluation text, AI score (0-100), and pass/fail.

### SectionProgress
Tracks whether a customer has completed a section (all lessons done).

### LessonProgress
Tracks individual lesson completion and video/audio resume position.

---

## Key Relationships

```
AdminUser --creates--> Token
Token --activates--> Customer (on first use)
Customer --has--> LessonProgress, SectionProgress, QuizAttempt

Course --has-many--> Section (ordered)
Section --has-many--> Lesson (ordered)
Section --has-one--> Quiz (polymorphic)
Course --has-one--> Quiz (final, polymorphic)

Quiz --has-many--> QuizQuestion (ordered)
QuizAttempt --belongs-to--> Quiz, Customer
QuizAttempt --has-many--> QuizAnswer
QuizAnswer --belongs-to--> QuizQuestion
```

---

## Section Unlock Rules

1. Section 1 is always unlocked
2. Section N unlocks when: all lessons in Section N-1 completed AND Section N-1 quiz passed
3. Final course quiz unlocks when: all sections completed AND all section quizzes passed

---

## Token Lifecycle

1. Admin generates token (code: VOCARE-XXXX-XXXX, optional label and expiry)
2. Customer enters token on login page
3. First use: customer enters name → Customer record created, token linked
4. Subsequent use: token looked up → existing customer loaded
5. Admin can revoke a token (soft-revoke via `revoked_at`)
